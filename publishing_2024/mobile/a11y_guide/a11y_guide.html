대체 텍스트

지침 내용 텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.

  • 텍스트 아닌 콘텐츠에 대한 대체 텍스트를 제공하지 않은 경우
  • 텍스트 아닌 콘텐츠에 대한 대체 텍스트를 부적절하게 제공한 경우
  • 사용자 콘텐츠 등록 시 대체 텍스트 입력 기능을 제공하지 않은 경우

1. 대체 텍스트가 제공되지 않은 콘텐츠

[메인 > 이벤트 배너]

  • 배너 이미지와 동등한 정보를 대체텍스트로 작성해야 합니다.(의미 없는 이미지 경우 alt="" 제공)

[신용대출 > 대출상품 > 상품소개 > 한도조회 - 약관 상세]

  • 약관 상세 이미지와 동등한 내용을 숨김 텍스트로 제공하고 aria-labelledby 속성으로 연결하여 임의 및 순차탐색 가능하도록 구현하거나,
    동등한 내용의 다운로드 파일을 제공해야 합니다.
  • 약관 상세 내용은 PDF를 다운로드하여 확인할 수 있으므로, 해당 영역에 다운로드를 유도하는 안내를 제공해야 합니다.

2. 대체 텍스트가 부적절한 컨트롤

[공통 > input 텍스트 입력]

  • 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 합니다.
    문구삭제[X], 입력된 텍스트 삭제[O]

초점

지침 내용 사용자 인터페이스는 초점이 적용되고, 논리적인 순서로 이동돼야 한다.

  • 초점이 접근되지 않거나 부적절한 콘텐츠
  • 초점의 순차적 이동순서가 논리적이지 않거나 일관성이 없는 콘텐츠
  • 초점 또는 키보드의 위치를 나타내는 개체가 시각적으로 표시되지 않은 경우
  • 숨겨진 콘텐츠 및 의미 없는 개체에 초점이 이동 되는 경우

1. 초점이 부적절하게 접근되는 콘텐츠

[신용대출 > 대출안내 > 간편한도조회(대출 안내 bottomSheet)]

  • 백그라운드 영역과 팝업 영역을 동등한 level이 되도록 하고
    팝업이 표시되면 백그라운드 영역에 aria-hidden 속성을 적용하여 모바일 스크린리더 초점이 팝업에 자동으로 이동되도록 구현합니다.
  • 레이어 팝업 종료 시 모바일 스크린리더 초점은 해당 레이어 팝업을 불러온 개체로 회귀되어야 합니다.

입력 도움

지침 내용입력서식은 입력 오류를 방지하거나 정정할 수 있어야 한다.

  • 입력 서식에는 1:1 대응하는 레이블을 제공해야 한다.
  • 입력 오류가 표시되어도 스크린리더 초점이 입력서식에 바로 이동되어 해당 오류를 인식할 수 없다면 접근성 오류로 진단한다.
  • 입력 오류 발생 시, 입력 오류에 대한 알림 및 정정 방법을 적절한 대체 텍스트로 제공한다.

1. 입력 오류에 대한 알림

[로그인 > 아이디 로그인]

  • 입력 오류 내용이 곧바로 인식되도록 aria-describedby="(오류 메세지 ID)" 속성을 추가합니다.

예측가능성

지침 내용화면전환이나 이벤트를 예측할 수 있어야 한다.

  • 사용자가 의도하지 않은 화면 전환이나 이벤트(새 창, 초점에 의한 맥락 변화 등) 등은 실행되지 않아야 한다.
  • 스크린 리더 사용자가 새 창 열림을 인지할 수 있도록 예측 가능하고 일관된 기능을 제공한다.

1. title="새 창", target="_blank"

title 속성, alert창, 레이어 팝업 등을 이용하여 화면 전환 기능을 사전에 고지하거나 사용자가 화면 전환 여부를 선택할 수 있도록 구현 합니다.

[상품소개 > 상담전화]

  • 전화번호 선택시 사전에 앱이 전환된다는 정보를 제공 한다.

[본인인증 > 휴대폰 본인확인 서비스동의]

  • 체크박스와 약관 상세 팝업 기능을 구분하여 제공하는 것이 가장 바람직하며, 불가능한 경우에는 최소한 스크린리더 사용자가 사전에 인지할 수 있도록 체크박스에 title="선택 시 상세보기 실행" 등의 속성을 제공해야 합니다.

보조기술과의 호환성

지침 내용사용자 인터페이스 컴포넌트는 보조 기술을 이용하여 사용할 수 있도록 해야 한다.

  • 유형 정보, 상태 정보는 컨텐츠를 파악하기 위한 중요정보로서 접근성에서는 이를 제공하지 않을 경우 접근성 오류로 진단한다.
  • 컨트롤 요소를 커스텀 UI로 제공하는 경우 유형 정보를 제공하지 않으면 스크린리더 사용자는 해당 컨트롤을 올바르게 이용할 수 없다.
  • 사용자가 컨텐츠를 제대로 파악하거나 탐색하기 위하여 유형 정보(머리말, 버튼, 링크, 이미지 등)와 상태 정보(선택됨, 확장됨 등)를 올바르게 제공해야 한다.

1. 유형 정보 role="button"

유형 정보가 제공되면 스크린리더 사용자가 콘텐츠를 올바르게 인식할 뿐만아니라 유형 정보 단위로 탐색을 해 탐색할 때 유용합니다.

[로그인 > 아이디 로그인]

  • input에 click 이벤트가 직접 제공되는 경우 TalkBack으로 실행할 수 없으므로 div 영역에 role="button"을 추가하여 버튼으로 인식될 수 있도록하고, click 이벤트를 추가하여 입력된 자릿수도 함께 제공해야 합니다.
  • aria-label="비밀번호 입력"이란 대체텍스트까지 제공하여 스크린리더가 컨트롤의 용도를 알 수 있도록 합니다.
  • 이 때, input은 aria-hidden="true" 속성을 사용하여 스크린리더 탐색에서 제외하여 줍니다.

2. 유형 정보 role="tab" (탭)

[본인인증 > (통신사)본인확인 이용약관 동의]

  • 선택된 컨트롤에 title="선택됨"과 같은 상태정보를 제공하거나, WAI-ARIA를 이용해 해당 컨트롤들이 Tab 목록으로 인식되도록 제공해야 합니다.

3. 유형 정보 role="radiogroup|radio|none" (라디오버튼)

[자동차대출 > 대출신청 > 바텀시트]

  • 여러개의 옵션 중 하나만 선택할 수 있다면 role="radio"를 추가하고 role="radiogroup"으로 그룹화 해줍니다.
  • aria-checked="true/false" 속성을 이용하여 컨트롤의 선택 상태에 따른 적절한 상태정보를 제공합니다.
  • 이때, <li>의 HTML 요소의 기본역할인 목록 항목 역할 의미를 role="none"을 사용하여 숨겨줍니다.

5. 상태 정보 aria-expanded="true/false" (아코디언)

버튼 선택시 영역이 '확장/축소'되는 경우 aria-expanded="true/fasle"를 사용하여 '확장됨/축소됨'등과 같이 상태 정보를 제공하여 줍니다.

[신용대출 > 약관]

  • 팝업 생성과 아코디언 기능을 함께 포함하고 있는 약관의 경우 title속성을 사용하여 팝업이 생성된 후 아코디언이 동작될 것임을 인지할 수 있도록 대체텍스트를 제공합니다.
  • 우측 <button>에는 aria-expanded="true/false" 속성으로 '확장됨/축소됨'등의 상태정보, 화면과 동등한 내용의 대체텍스트를 title로 함께 제공해야 합니다.

[자동차대출 > 자동차리스 > 견적만들기]

  • 상위의 <div>에 aria-expanded="true/false" 속성으로 '확장됨/축소됨'등의 상태정보를 제공해야 합니다.
  • <div>에 role="button" 속성을 추가하여 하나의 버튼 콘텐츠로 만들고, title 속성으로 기능정보까지 함께 제공합니다.
  • 우측 <button> 에는 aria-hidden="true"를 제공하여 인식 및 접근되지 않도록 구현해야 합니다.

7. 커스텀 select

[회원가입 > 휴대폰 인증]

상태정보가 제공되지 않은 컨트롤에

  • <button>, <select> 요소가 겹쳐진 상태로 제공되는 경우 스크린리더 임의탐색으로는 버튼에만 접근되어 <select>에는 aria-hidden="true" 속성을 제공하여 초점이 접근되지 않도록 해야 합니다.
  • 실제로 팝업 기능이 제공된 <button>에 aria-label="KT" 등 현재 선택 항목을 제공하여 초점 접근 시 함께 인식되도록 합니다.
  • 바텀시트 팝업 내 버튼에 title="선택됨"을 추가하여 선택 상태정보를 제공합니다.
