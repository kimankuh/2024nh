규칙

  1. 1. 기획서에 있는 화면ID가 HTML 파일명이 된다
  2. 2. 파일명은 'OO-0000-OO' 형식으로 각 카테고리에 따라 정한다
  3. 3. 화면 아이디 정의 방식은 아래와 같다.
    • 카테고리 기준
    • 숫자 기입. 단, 1자리 숫자는 페이지 사이에 생길 추가 페이지를 위해 남겨두고 순차적으로 진행
    • 페이지의 유형대로 작성한다
  4. 4. 메뉴구조도 화면ID.html로 구성
  5. 5. 이후 관리는 워크리스트에 순차적으로 정리한다

  • 두 줄이상일 경우 범위로 표시
  • 한 줄일 경우 해당 선언 마지막 뒤에 이름과 같이 표시

  • 주석 첫 부분에 '수정 [년도.월.일]' 입력 후 내용 기재. 날짜 사이는 점('.')으로 표시하며, 담당자들의 주석 규칙은 작업자 모두 일치 해야한다
    ※ 수정 날짜를 찾거나 개발자에게 안내할때 유니크한 부분이 있어야 쉽게 'Search' 할 수 있기 때문에 대괄호('[]') 를 추가 함

이미지(Images)
  • 각자 기호와 용도에 맞게 이미지 작업, 관리한다. 하지만 이행할 경우 모든 퍼블 소스, 파일을 문서로 관리하는 경우가 생기는 상황을 고려하여 이미지를 생성한다
  • 기본 적인 이미지 파일명은 언더바('_')로 관리하며 다음과 같은 규칙을 순서로 가능한 선에서 생성하도록 한다.
    1. 정체성 :
      img, ico 3글자로 통일하되, 관리차원에서 다른 분류가 필요하다면 공유하여 네이밍의 정확성을 더한다
    2. 사용처(생략가능) :
      insurance, capital, card 등
    3. 이름 :
      편하게 이름 짓기
    4. Ex) img_card_friends, ico_capital_arrow, ico_linkarrow
  • 이미지(img)가 의미가 있는 경우 HTML <img> 사용 후 alt="" 속성으로 기입하여 접근성을 대비한다
  • 아이콘(ico)은 CSS로 사용한다
  • 에니메이트와 같은 엑션이 들어갈 경우 해당 페이지만 사용해야 하기 때문에 페이지 아이디를 더하여 새로운 네이밍을 한다.
    페이지 아이디만 앞에 기재 되어 있다면 뒤에는 자유롭게 네이밍 한다
    Ex)D14500_00_bg, D14500_00_img01~X, D14500_01_img01_2

양식 통일
  • checked , readonly , disabled 등등 속성은 간결하게 사용한다.
  • href 속성값은 "#"을 금지하고 "#none" 으로 사용한다.

	<input type="checkbox" checked=""> (x)
	<input type="checkbox" readonly="readonly"> (x)
	<input type="checkbox" disabled> (o)

	<a href="#"</a> (x)
	<a href="#none"</a> (o)

마크업(Mark up)

Doctype

DOCTYPE은 html5 버전을 선언


	<!DOCTYPE html>
META Content

iPhone X 등의 Notch를 가지고 있는 기종의 스크린사이즈 조절을 위해
"viewport-fit=cover" 추가


	<meta name="viewport" content="
		width=device-width,
		initial-scale=1.0,
		maximum-scale=1.0,
		minimum-scale=1.0,
		user-scalable=no,
		viewport-fit=cover
	">
HEADER

header는 html5 semantic code를 사용하여 header 태그를 사용하며, 항상 클래스 header를 포함


	<!-- header 시작 -->
	<header class="header">
	</header>
	<!-- header 끝 -->

단, 풀팝업(FULL POP-UP) 및 팝업(POP-UP)의 경우 div 태그에 header 클래스를 사용하여 페이지 header와 차별


	<!-- header 시작 -->
	<div class="header">
	</div>
	<!-- header 끝 -->
NAV

상단 공통 네비게이션 메뉴는 header 태그 안에 nav로 선언


	<!-- header 시작 -->
	<header class="header">
		<!-- nav 시작 -->
		<nav role="presentation">
			<li>
				<a href="#none" class="back-btn">뒤로가기</a>
			</li>
			<h1 class="header-title"><a href="#none">e금리우대적금</a></h1>
			<li>
				<a href="#none" class="home-btn">뒤로가기</a>
			</li>
		</nav>
		<!-- nav 끝 -->
	</header>
	<!-- header 끝 -->
CONTENTS

상단 공통 네비게이션 메뉴는 header 태그 안에 nav로 선언

차후 추가 예정

Gap Class

컨텐츠와 컨텐츠의 간격은 디자인에 따라 변동이 되고 일관성을 유지하기 어려워 .gap 클레스를 사용. 이로 인해 이격 거리를 한번에 조율할 수 있음

.gap-xs :2rem (20px) / 디자인 기준 (40px)
.gap-sm :3rem (30px) / 디자인 기준 (60px)
.gap :4rem (40px) / 디자인 기준 (80px)
.gap-lg :6rem (60px) / 디자인 기준 (120px)

※ 큰 div(wrapper, area) 와 같은 컴포넌트 만 추가해야 함.
작은 단위의 오브젝트와 section tag에는 사용하지 않는다


	<div class="box"> 
		<div class="text gap">{x}
			컨텐츠 내용 
		</div>
	</div>
	<section class="box gap"> 내용 </section>{x}
<div class="box gap"> 컨텐츠 내용 </div>{o}

CSS(Cascading Style Sheets)

  • Single-line 코드로 작성한다
  • !important 사용 자제한다
  • iframe으로 불어오는 페이지가 많이 있어서 전체 선택자(*) 사용을 자제
  • 색상 스타일 선언시 축약형을 사용한다
  • 가급적 스타일 속성 뒤 ;은 띄어쓴다
  • 가급적 보더 속성과 관련된 내용에는 'none' 이 아닌 '0' 으로 통일한다
  • 폰트는 Regular, Medium 총 2가지 이며, 기본은 Regular 로 통일 되어 있다.
    폰트 굵기 선언 시 'normal', 'bold' 로 굵기를 조정한다
  • 폰트 스타일 Regular : font-weight:normal / Medium : font-weight:bold

	.txt{color:#333333;} (x)
	.txt{color:#333;} (o)

	.box {width: 100px100px; height: 100px;} (x)
	.box{width:100px; height:100px;} (o)

	.bord{border:none} (x)
	.bord{border:0} (o)

	.txt{font-weight:400, 700} (x)
	.txt{font-weight:normal, bold} (o)
CSS 선언 순서

스타일 선언에 규칙을 부여하여 중복된 스타일을 쓰지 않도록 하는 방향으로 다음과 같이 순서를 정한다

  1. 0. content (가상클레스일 경우)
  2. 1. display
  3. 2. position
  4. 3. width, height
  5. 4. padding, margin
  6. 5. border
  7. 6. font-size, weight, color
  8. 7. text-algin, indent
  9. 8. Background
  10. 9. Effect 관련 (shadow, animate 등등)
  11. ETC. 그 외
:root Style
  • 수정 시 다수 케이스가 같이 변경되어야 하는 경우
  • 최종 결정되지 않아 변동이 한번에 적용되어야 하는 경우
  • 디자인팀과 협의 후 스타일이 일치하여 변동되지 않지만, 작업자 마다 값이 틀릴 수도 있는 경우 대비

위의 상황을 고려하여 :root 를 common.css 상단에 선언하여 사용한다 단, 선언하고자 하는 값과 일치한다고 해도 반드시 목적에 맞게 사용한다.
또한 :root에 선언 시 Camelcase 를 사용한다

--paddCommon 2.4rem
padding:0 var(--paddCommon)
컴포넌트의 양 옆에만 사용
--paddBig 3.2rem

컴포넌트의 양 옆에만 사용
--shadowStyle .5rem .5rem 2rem 0 rgba(0,0,0,.14)
box-shadow:var(--shadowStyle)
